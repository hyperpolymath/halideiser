== What Is This?

Halideiser compiles image and video processing pipelines to optimised
**Halide schedules** — achieving 10-100x speedups over hand-tuned C.

Halide separates the algorithm from the schedule (how it runs on hardware).
Halideiser makes this separation automatic.

== How It Works

Describe your image pipeline in `halideiser.toml`. Halideiser:

1. Parses your pipeline stages (blur, sharpen, resize, convolve, etc.)
2. Generates Halide algorithm definitions
3. Auto-tunes the schedule for your target hardware (CPU/GPU/SIMD)
4. Creates a Zig FFI bridge for calling from any language

== Key Value

* **10-100x faster image/video processing**
* **Automatic hardware scheduling** — SIMD, GPU, multi-core
* **No Halide expertise needed** — describe the pipeline, get the speed

== Architecture

Follows the hyperpolymath -iser pattern (same as https://github.com/hyperpolymath/chapeliser[Chapeliser]):

* **Manifest** (`halideiser.toml`) — describe WHAT you want
* **Idris2 ABI** (`src/abi/`) — formal proofs of interface correctness
* **Zig FFI** (`ffi/zig/`) — C-ABI bridge to target language runtime
* **Codegen** (`src/codegen/`) — generates target language wrapper code
* **Rust CLI** — parses manifest, validates, generates, builds, runs

User writes zero target language code. Halideiser generates everything.

Part of the https://github.com/hyperpolymath/iseriser[-iser family] of acceleration frameworks.
